# Collaborative Trip Planning Backend

## Overview
A production-grade, modular backend for a collaborative trip planning platform.
Built with: Express.js, TypeScript, Prisma, PostgreSQL (Neon DB), Bun.

## Architecture
Strict layered architecture with the following pattern:
- **Modules**: Feature-based folders containing controllers, services, repositories, routes, DTOs, and validation schemas.
- **Common**: Shared errors, responses, and permission logic.
- **Middlewares**: Auth, error handling, validation, and security.
- **Utils**: Reusable helper functions.
- **Config**: Database, CORS, and Swagger settings.

## Database Schema
10+ tables including:
- `User`: Profile and authentication.
- `Trip`: Central entity for planning.
- `TripMember`: Junction for user roles in trips (OWNER, EDITOR, VIEWER).
- `Day` & `Activity`: Itinerary structure.
- `Expense` & `ExpenseParticipant`: Budgeting and split logic.
- `Comment`, `Checklist`, `Attachment`, `Reservation`: Collaboration tools.
